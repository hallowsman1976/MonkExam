# ระบบบันทึกการตรวจสุขภาพพระสงฆ์ (Monk Health Check Record System)

ระบบสำหรับบันทึกข้อมูลการออกตรวจสุขภาพพระสงฆ์ในชุมชน โดยใช้ **Google Apps Script** เป็น Backend API และ **Google Sheets** เป็นฐานข้อมูล รองรับการใช้งานบนมือถือ 100%

## ไฟล์ในโปรเจกต์
1. `Code.js`: (หรือ Code.gs) มีฟังก์ชันสำหรับการทำงานในฝั่ง Server (Backend API, จัดการ Google Sheets)
2. `index.html`: โค้ดหน้าเว็บ Frontend (HTML/TailwindCSS/JS)

---

## 🛠️ วิธีการติดตั้งและการ Deploy เป็น Web App

### ขั้นตอนที่ 1: การนำโค้ดไปใส่ใน Google Apps Script
หากคุณใช้ Clasp อยู่แล้ว สามารถใช้คำสั่ง `clasp push` เพื่อนำโค้ดขึ้นไปบน Google Apps Script ได้เลย
หากทำแบบ Manual:
1. สร้าง Google Sheets เปล่าบน Google Drive
2. ไปที่เมนู `ส่วนขยาย (Extensions)` > `Apps Script`
3. ก๊อปปี้โค้ดในไฟล์ `Code.js` ไปวางทับในไฟล์ `Code.gs` ของหน้า Apps Script
4. กดปุ่ม `+` เพิ่มไฟล์ HTML ตั้งชื่อว่า `index` แล้วก๊อปปี้โค้ดจาก `index.html` ไปวาง

### ขั้นตอนที่ 2: การสร้างโครงสร้างฐานข้อมูล (Google Sheets)
1. ในหน้าต่าง Apps Script ไปที่ไฟล์ `Code.gs`
2. เลือกฟังก์ชันชื่อ `setupSheets` บนแถบเครื่องมือด้านบน
3. กดปุ่ม `เรียกใช้ (Run)` (อาจมีการขอสิทธิ์การเข้าถึง Google Account ให้กดอนุญาตและ Continue)
4. เมื่อรันเสร็จ กลับไปดูที่หน้า Google Sheets จะพบว่ามีการสร้าง Sheet `HealthRecords` และ `Settings` พร้อมใส่ Header อัตโนมัติ

### ขั้นตอนที่ 3: การ Deploy เป็น Web App
1. มุมขวาบนในหน้า Apps Script กดปุ่ม `การทำให้ใช้งานได้ (Deploy)` > `การทำให้ใช้งานได้รายการใหม่ (New deployment)`
2. กดไอคอนรูปเฟือง ⚙️ และเลือกประเภท `เว็บแอป (Web App)`
3. ตั้งค่าดังนี้:
   - **คำอธิบาย:** "ระบบตรวจสุขภาพพระสงฆ์ V1"
   - **เรียกใช้ในฐานะ:** เลือก "ฉัน (อีเมลของคุณ)"
   - **ผู้มีสิทธิ์เข้าถึง:** เลือก "ทุกคน (Anyone)"
4. กดปุ่ม `ทำให้ใช้งานได้ (Deploy)`
5. เมื่อ Deploy เสร็จสิ้น ให้ก๊อปปี้ **URL เว็บแอป (Web App URL)** เอาไว้

### ขั้นตอนที่ 4: เชื่อมต่อ Frontend กับ Backend API
1. กลับมาที่ไฟล์ `index.html` เลื่อนลงไปที่ส่วนของ `<script>` บรรทัดที่ประมาณ 362
2. หาบรรทัดนี้: `const SCRIPT_URL = "";`
3. นำ URL ที่ได้จากข้อ 3 มาใส่ เช่น `const SCRIPT_URL = "https://script.google.com/macros/s/xxxx/exec";`
4. หากแก้ไขใน Apps Script ให้บันทึกไฟล์
5. **สำคัญ:** หากมีการแก้ไขโค้ดใดๆ ต้องกด `Deploy` > `จัดการการทำให้ใช้งานได้ (Manage deployments)` แล้วเลือกเวอร์ชันใหม่เสมอ

## 🚀 วิธีการใช้งาน

1. นำ URL ของ Web App ไปเปิดในเบราว์เซอร์บน PC หรือมือถือ (แนะนำ Chrome/Safari)
2. หน้าเว็บจะแสดง Dashboard เบื้องต้น
3. ไปที่เมนู **"บันทึกข้อมูลใหม่"** เพื่อกรอกข้อมูล เมื่อกดบันทึก ข้อมูลจะวิ่งไปลงที่ Google Sheets ทันที
4. ไปที่เมนู **"ทะเบียนข้อมูล"** เพื่อค้นหา ดูข้อมูล แก้ไข ลบ (เปลี่ยนสถานะ) หรือดึงข้อมูลออกเป็นไฟล์ CSV (Export CSV) 
   - *หมายเหตุ: การ Export CSV จะทำการปิดบังเลขบัตรประชาชนเป็น xxxxxxxxx1234 อัตโนมัติตามหลัก PDPA*

## 🗄️ โครงสร้างฐานข้อมูล (Google Sheets)

### 1. Sheet: HealthRecords
เก็บข้อมูลการตรวจสุขภาพทุกรายการ:
1. `RecordID`: รหัสรายการอ้างอิง
2. `Timestamp`: เวลาที่บันทึกข้อมูล
3. `ชื่อวัด` ถึง `เบอร์โทรผู้สำรวจ`: ข้อมูลพื้นที่และเจ้าหน้าที่
4. `ชื่อ-สกุล` ถึง `เบอร์โทรพระสงฆ์`: ข้อมูลทั่วไปของพระสงฆ์
5. `น้ำหนัก`, `ส่วนสูง`, `BMI`, `แปลผล BMI`: ข้อมูลน้ำหนัก-ส่วนสูง
6. `ความดันตัวบน SBP`, `ความดันตัวล่าง DBP`, `แปลผลความดันโลหิต`: ข้อมูลความดัน
7. `สถานะการตรวจน้ำตาล`, `ผลน้ำตาลปลายนิ้ว DTX`, `แปลผล DTX`, `คำแนะนำ DTX`: ข้อมูลเจาะเลือด
8. `โรคประจำตัว`, `หมายเหตุ`, `ผู้บันทึก`: ข้อมูลอื่นๆ
9. `สถานะข้อมูล`: สถานะว่า ข้อมูลยังใช้งานได้ (active) หรือถูกลบ (inactive)

### 2. Sheet: Settings
ใช้สำหรับเก็บการตั้งค่าทั่วไปของระบบ เช่น ชื่อระบบ หรือเขตเวลา

## 🔌 API Endpoints
ระบบทำงานบน `doGet` และ `doPost` หากต้องการนำ API ไปเชื่อมต่อกับระบบอื่น:

### GET Requests (`?action=...`)
- `action=healthCheck` : ตรวจสอบสถานะการเชื่อมต่อ API
- `action=dashboard` : คืนค่าสรุปผลข้อมูลรวม (จำนวนวัด, จำนวนรูป, ความเสี่ยงแต่ละประเภท)
- `action=listRecords` : คืนค่าข้อมูลประวัติการตรวจทั้งหมดที่เป็น active
- `action=getRecord&id=xxx` : ดึงข้อมูลรายบุคคลตาม `RecordID`
- `action=search&keyword=xxx` : ค้นหาประวัติการตรวจตาม Keyword (ชื่อ, วัด, ตำบล)

### POST Requests (JSON Payload พร้อม `"action": "..."`)
- `action=createRecord` : บันทึกข้อมูลใหม่
- `action=updateRecord` : แก้ไขข้อมูล โดยต้องแนบ `RecordID` มาด้วย
- `action=deleteRecord` : ลบข้อมูล (เปลี่ยนสถานะเป็น inactive) โดยส่งแค่ `{ "action": "deleteRecord", "RecordID": "xxx" }`

**Format การตอบกลับ (Response Format):**
```json
{
  "success": true,
  "message": "ดำเนินการสำเร็จ",
  "data": { ... }
}
```
